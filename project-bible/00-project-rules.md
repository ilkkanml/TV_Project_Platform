# Project Rules

## General Rules

- Follow the approved folder tree.
- Do not change architecture without explicit instruction.
- Keep all progress additive.
- Keep platform code modular.
- Keep shared types in `packages/shared`.
- Keep production secrets outside the repository.
- Do not add copyrighted content, stream sources, channel lists, or provider data.

## Legal Boundary Rules

- Do not make the backend a stream provider.
- Do not make the backend a playlist provider.
- Do not store playlist data as source of truth.
- Do not add CDN, relay, or transcoding responsibilities.
- Do not market the project as a content access service.

## Development Rules

- Prefer small, reviewable changes.
- Keep JSON pretty-printed with 2 spaces.
- Keep Markdown readable.
- Keep YAML multiline and human-editable.
- Keep environment values as placeholders in examples.
